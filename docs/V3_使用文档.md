# NetSpider-Max v3 — 弱口令检测工具使用文档

## 一、工具概述

**NetSpider-Max v3** 是一个全方位弱口令/默认凭据检测工具，用于网络安全审计、等保合规和资产梳理。

### 核心能力

| 能力 | 说明 |
|------|------|
| **协议覆盖** | 22 种协议：SSH, Telnet, FTP, HTTP, RDP, SMB, MySQL, PostgreSQL, MSSQL, Oracle, MongoDB, Redis, Elasticsearch, LDAP, VNC, SNMP, WinRM, RTSP, SMTP, IMAP, POP3, IPMI |
| **资产识别** | Nmap -sV -O 服务指纹识别，支持 OS/主机名/产品版本提取 |
| **双驱动引擎** | asyncio+httpx (2000+ HTTP并发) + ThreadPoolExecutor (200线程原生协议) |
| **统一资产表** | 一张 JSON 表驱动所有凭据查询，不改代码即可扩展 |
| **设备专属检测** | H3C / 华为 / 思科 Web 管理界面默认口令检测 |
| **未授权访问** | 自动检测 Redis/MongoDB/FTP/SNMP/VNC/Memcached/ES 未授权 |

### 目录结构

## 二、快速开始

### 环境要求
- Python 3.10+
- Nmap (可选，用于服务发现，不可用时回退到内置TCP扫描)

### 安装依赖
```bash
# 核心依赖（必选）
pip install paramiko        # SSH 协议

# 可选依赖
pip install httpx           # 异步HTTP (2000+并发)
pip install beautifulsoup4  # 智能表单解析
pip install oracledb        # Oracle 凭据验证
pip install pymssql         # MSSQL 凭据验证
pip install pymongo         # MongoDB 凭据验证
```

### 启动
```bash
# GUI 模式
python main.py

# CLI 模式
python main.py --cli 192.168.1.0/24

# CLI 自定义端口
python main.py --cli 10.0.0.0/16 -p 22,80,443,3306,3389,8080

# 查看版本
python main.py --version
```

### 验证工具完整性
```bash
python verify_weakpass.py
```

## 三、核心概念：统一资产弱口令库

**`data/unified_asset_table.json`** 是整个工具的**唯一数据源**。

工具代码永远不需要修改，新增检测能力只需在 JSON 中添加条目。

### JSON 结构

```json
{
  "_meta": { "version": "1.0", "name": "统一资产弱口令库" },

  "top100": [
    {"u": "admin", "p": "admin"},
    {"u": "admin", "p": "123456"}
  ],

  "unauthorized": [
    {"service": "redis", "desc": "Redis未授权访问",
     "creds": [{"u": "", "p": ""}]}
  ],

  "service_ports": {
    "ssh": [22], "http": [80, 443, 8080, ...], ...
  },

  "assets": [
    {
      "category": "网络设备",
      "subcategory": "路由器",
      "vendor": "h3c",
      "display": "H3C",
      "keywords": ["h3c", "secpath", "comware", "msr"],
      "models": ["MSR 2600/3600/5600"],
      "services": {
        "http": {
          "credentials": [
            {"u": "admin", "p": "admin", "scope": "通用Web管理"},
            {"u": "admin", "p": "adminer3100", "scope": "ER 3100"}
          ]
        },
        "ssh": {
          "credentials": [
            {"u": "admin", "p": "admin"},
            {"u": "", "p": "", "scope": "Console默认"}
          ]
        }
      }
    }
  ]
}
```

### 扫描器如何匹配

扫描器发现一个资产时，按以下顺序组装凭据列表：

```
Tier 1: TOP 100（100条，始终最前）
    ↓
Tier 2: 未授权访问（按 service 匹配）
    ↓
Tier 3: 厂商关键字匹配（keywords 模糊匹配 product/hostname/os）
    ↓
Tier 4: 规则变异（可选，hostname+年份等）
    ↓
去重 → 最多300条 → 执行爆破
```

## 四、如何添加新资产

### 4.1 添加一个新的厂商设备

在 `assets` 数组中追加一条：

```json
{
  "category": "网络设备",
  "subcategory": "防火墙",
  "vendor": "newvendor",
  "display": "新厂商",
  "keywords": ["newvendor", "nv-fw", "nv-os"],
  "models": ["NV-FW 1000", "NV-FW 2000"],
  "services": {
    "http": {
      "credentials": [
        {"u": "admin", "p": "admin", "scope": "Web管理"},
        {"u": "admin", "p": "NewVendor@123", "scope": "默认密码"}
      ]
    },
    "ssh": {
      "credentials": [
        {"u": "admin", "p": "admin"}
      ]
    },
    "snmp": {
      "credentials": [
        {"community": "public"},
        {"community": "private"}
      ]
    }
  }
}
```

### 4.2 关键字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `vendor` | ✅ | 内部标识，唯一 |
| `display` | ✅ | 显示名称 |
| `keywords` | ✅ | 用于从 Nmap 扫描结果中匹配该厂商。扫描器将 `product` + `hostname` + `os_family` 合并为字符串，与 keywords 逐一比对 |
| `models` | 否 | 影响的型号列表（仅文档用途，不参与匹配） |
| `services.{type}.credentials` | ✅ | 该服务类型的凭据列表。`u`=用户名，`p`=密码，`community`=SNMP社区字符串 |
| `scope` | 否 | 凭据适用范围说明（仅文档用途） |

### 4.3 Keywords 编写建议

`keywords` 是匹配的关键。扫描器拿到 Nmap 的产品名（如"H3C SecPath F1000"），在 keywords 中查找子串。建议：

- 包含**厂商名**（如 `"h3c"`）
- 包含**产品线代号**（如 `"secpath"`, `"msr"`, `"er-"`）
- 包含**操作系统特征**（如 `"comware"`, `"vrp"`）
- 全部**小写**（匹配时不区分大小写）
- 不要太短（如单个字母"a"会误匹配）

### 4.4 添加新的服务端口

如果需要在默认扫描中覆盖新端口，在 `service_ports` 中添加：

```json
"service_ports": {
  "http": [80, 443, 8080, 8443, 你的新端口],
  "新服务类型": [新端口1, 新端口2]
}
```

## 五、未授权访问检测

以下服务类型会自动检测未授权访问（无需匹配厂商）：

| 服务 | 检测方式 | 凭据 |
|------|---------|------|
| Redis | 发送 PING，收 PONG = 无密码 | `(空):(空)` |
| MongoDB | 尝试无认证连接 | `(空):(空)`, `admin:(空)` |
| FTP | 匿名登录 | `anonymous:anonymous`, `ftp:ftp` |
| SNMP | 默认 community | `public`, `private` |
| VNC | 无密码认证 | `(空):(空)` |
| Memcached | 默认无认证 | `(空):(空)` |
| Elasticsearch | 旧版无认证 | `(空):(空)` |

## 六、CLI 参数

```
python main.py --cli <CIDR> [参数]

参数:
  -p, --ports    端口列表，逗号分隔 (默认: 22,80,443,3306,3389,5432,6379,8080,8443,27017)
  -t, --threads  线程数 (默认: 100)
  --timeout      连接超时秒数 (默认: 2.0)
  --no-nmap      不使用 Nmap (回退内置 TCP 扫描)
  -o, --output   输出文件前缀
  --dict         自定义字典路径 (不使用统一表时)
```

### 示例
```bash
# 扫描 C 段，全端口
python main.py --cli 192.168.1.0/24 -p 1-65535

# 扫描 /16，只测 SSH/RDP/MySQL
python main.py --cli 10.0.0.0/16 -p 22,3389,3306 --no-nmap

# 导出结果
python main.py --cli 192.168.1.0/24 -o scan_result
# 输出: scan_result.json
```

## 九、扩展开发

### 添加新协议插件

1. 在 `netspider/protocols/` 创建检测器函数 `test_xxx(ip, port, user, pwd) -> bool`
2. 在 `netspider/protocols/__init__.py` 的 `TESTER_MAP` 中注册
3. 在 `netspider/plugins/wrappers.py` 的 `_register_all()` 中注册插件
4. 在 `tests/mock_servers.py` 中添加模拟服务端
5. 运行 `verify_weakpass.py` 确认通过

### 添加设备专属 Web 插件

1. 在 `netspider/plugins/web/` 创建插件类
2. 继承 `BasePlugin`，覆盖 `test()` 和 `test_async()`
3. 注意：`is_async = True` 的插件走 asyncio 通道（需要 httpx）
